ML Pipeline

Step-by-step methodology — from raw sensor data to defect prediction

01

Data Ingestion & Preprocessing

49 process parameters from hot rolling mill sensors loaded. Median imputation applied to 12 columns with missing values. QuantileTransformer applied for Logistic Regression branch. All transformers fit on training data only — no leakage.

SimpleImputerQuantileTransformerNo Data Leakage
02

Feature Engineering — 49 → 114 Features

Global statistics: mean, std, max, min, range, CV, IQR, skewness, kurtosis. Stage-wise aggregates per rolling stage (5 stages × 5 stats). Inter-stage deltas. Squared and log transforms of top-signal features (X35, X13, X36, X34, X10). Pairwise ratios.

Global StatsStage AggregatesInter-stage DeltasTop-signal Transforms114 Features
03

Mutual Information Feature Selection

Mutual Information scores computed between all 114 features and the target label. Top 60 features selected — eliminating noise while retaining maximum signal. Selection is fit on training data only.

Mutual InformationTop 60 SelectedNoise Reduction
04

Class Imbalance — 292× Cost Weighting

Severe class imbalance: only 4.9% defect rate (66 / 1,352). All 6 ensemble members trained with class_weight = {defect: 292} — making each missed defect 292× more costly than a false alarm. Ensures the models never ignore the minority class.

Class Weight 292×Cost-Sensitive Learning4.9% Defect Rate
05

6-Model Ensemble + Rank-Based Threshold

RF, ET, XGB, LGB, GB, LR probability scores averaged. 5-fold stratified cross-validation used to find OOF scores. Final threshold = score of the (expected_N × 1.3)th highest coil — providing a 30% safety buffer. Guarantees Recall = 100% on training data.
